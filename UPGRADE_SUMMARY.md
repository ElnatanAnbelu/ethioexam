# UPGRADE_SUMMARY.md — Grade logic fix + 4,000-question expansion

## What changed
1. **Grade-aware architecture.** Converted the curated bank to a **compact row format** (short keys
   `{i,g,u,d,t,q,o[4],c,e}` + a defensive `hydrate()`), re-tagged the original 408 questions to the
   **official MoE unit taxonomy** (Grades 9–12), added a `UNIT_TAXONOMY` constant, a **grade filter**
   (All / 9 / 10 / 11 / 12) in Free Practice (Mock Exam still mixes all grades), and a **grade-grouped
   unit picker** that shows every official unit per grade with its live question count (0-count units
   dimmed so gaps are visible).
2. **Massive verified expansion.** Generated **~3,400 new questions** via a generate → independent
   re-solve → keep-only-verified pipeline (16 generate + 16 verify agents per grade×difficulty wave per
   subject), strictly aligned to the Ethiopian MoE new curriculum, weighted G11/G12 ≈ 35% each, G10 ≈ 20%,
   G9 ≈ 10%, difficulty ≈ easy 30 / medium 35 / hard 25 / impossible 10.

## Quality gate
- **Two-stage gate:** every item was generated by one agent and then **independently re-solved by a
  different agent**, which dropped/corrected anything it couldn't confirm (a wrong key is worse than fewer
  questions). Per-subject verify-drop + dedupe rates were reported each run.
- **Orchestrator hand-check:** 12 hardest Physics + 12 hardest Math items re-computed by hand — **24/24
  correct** (kinematics, energy loss in inelastic collisions, Young's modulus, orbital speed, limits,
  integrals, cross products, Aⁿ of a triangular matrix, Σk/3ᵏ, population SD, etc.).
- **Ethiopian-curriculum fit:** items use official MoE unit names, SI units, and natural Ethiopian context
  (Adama/Awash/Blue Nile, teff/injera/shiro, hydropower, *Anopheles*/malaria, sugar & cement industries).

## Caveats / honest notes
- These are AI-generated-then-AI-verified items. The two-stage gate + hand-sample passed cleanly, but this
  is a *practice* bank — for a few thousand items, expect the occasional imperfect distractor or phrasing
  the curated 408 starter set didn't have. Spot-check as you use it; fixing any item is a one-line edit in
  `banks/qb-<subject>.js` (then re-inline).
- Near-duplicate questions are deduped by normalized question text (first 80 chars). A small number of
  same-concept-different-numbers items remain by design (good for drilling).
- File size grew to ~1.5 MB. Still a single self-contained file per the hard constraint; comfortably
  loadable on iPad Safari. If it ever needs to grow much larger, consider lazy-loading per subject — flagged,
  not yet needed.

## How to extend further
Edit `banks/qb-<subject>.js` (compact rows, see `FORMAT.md`), then re-inline into `EthioExam.html`'s
`window.QB_<SUBJECT>` array and re-run `node --check` + the hydration count. The generate/verify workflow
script used for this pass is reusable (parameterized by subject via `args`).
